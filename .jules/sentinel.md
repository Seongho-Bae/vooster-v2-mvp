## 2024-05-24 - Path Traversal Vulnerability in `exportGherkin`
**Vulnerability:** A path traversal vulnerability existed in the `exportGherkin` command, allowing arbitrary files to be created outside the project root if a malicious path string (e.g., `../../../../tmp/pwned.txt`) was passed via the `--output` argument. The vulnerability was caused by blindly joining the user-provided output string with the configuration root without enforcing directory boundaries.
**Learning:** `join` alone does not secure file paths against `../` attacks. In a local-first application built around a designated directory (like `.vspec` projects), inputs resolving to relative output files must always be bounded.
**Prevention:** Construct absolute paths using `path.resolve(root, target)` and verify boundaries using `const rel = path.relative(root, target)`. Always check both `rel.startsWith("..")` and `path.isAbsolute(rel)` to ensure security across all platforms, including Windows. Throw a specific `VspecError` (e.g. `INVALID_ARGUMENT`) when path traversal is detected.
